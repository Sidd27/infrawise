# S3 Buckets + Event Notifications Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add S3 bucket + event notification extraction so Claude Code knows bucket names, event configs, and which Lambdas they trigger.

**Architecture:** New `src/adapters/aws/s3.ts` adapter → `ServicesMeta.s3` → `buildGraph` creates bucket nodes and back-propagates S3 notifications as Lambda triggers → three analyzers → `get_s3_overview` MCP tool. Also adds `verify` as a first-class Finding severity.

**Tech Stack:** `@aws-sdk/client-s3` (new), vitest, existing patterns throughout.

**Spec:** `docs/superpowers/specs/2026-06-02-s3-adapter-design.md`
